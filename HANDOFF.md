# Передача проекта знакомому

Пошаговая инструкция: кто что делает, как задеплоить на Render и передать готовый бот.

## Роли

| Кто | Что делает |
|-----|------------|
| **Знакомый** | Создаёт Telegram-бота через BotFather, добавляет бота в рабочий чат, даёт доступ к GitHub/Render (у вас уже есть) |
| **Вы** | Пишете код, заливаете на GitHub, настраиваете Render, почту, тестируете и передаёте готовый результат |

## Этап 1 — Подготовка (вы)

### 1.1. Репозиторий на GitHub

```bash
git init
git add .
git commit -m "Initial: OLX email bot for Render"
git branch -M main
git remote add origin https://github.com/<аккаунт-знакомого>/kirs-bot.git
git push -u origin main
```

**Важно:** файл `.env` в git не попадает (он в `.gitignore`). Секреты хранятся только на Render.

### 1.2. Локальная проверка (опционально)

Перед деплоем можно проверить у себя:

```bash
python -m venv venv
venv\Scripts\activate
pip install -r requirements.txt
copy .env.example .env
# заполнить .env тестовыми данными
python main.py
```

Локальный запуск **не нужен для продакшена** — бот будет работать на Render.

## Этап 2 — Telegram (знакомый + вы)

### 2.1. Знакомый создаёт бота

1. Открыть [@BotFather](https://t.me/BotFather)
2. `/newbot` → придумать имя и username
3. Скопировать **токен** и передать вам (не публиковать в GitHub!)

### 2.2. Знакомый добавляет бота в чат

- Создать группу для менеджеров **или** использовать личный чат
- Добавить бота в группу
- Выдать боту право писать сообщения (для групп)

### 2.3. Вы узнаёте chat_id

Временно задайте токен и запустите утилиту:

```bash
# в .env только TELEGRAM_BOT_TOKEN
python scripts/get_chat_id.py
```

Знакомый пишет боту в нужном чате — скрипт выведет `chat_id`.  
Этот ID понадобится на Render.

## Этап 3 — Почта (вы)

### 3.1. Единый ящик

На почте знакомого (доступ у вас есть):

1. Включить **IMAP** (Gmail: Настройки → Пересылка и POP/IMAP)
2. Создать **пароль приложения** (не обычный пароль!)
   - Gmail: https://myaccount.google.com/apppasswords
3. Настроить пересылку уведомлений OLX с разных аккаунтов на этот ящик

### 3.2. Данные для Render

| Переменная | Значение |
|------------|----------|
| `EMAIL_HOST` | `imap.gmail.com` (или хост другого провайдера) |
| `EMAIL_PORT` | `993` |
| `EMAIL_USER` | адрес единого ящика |
| `EMAIL_PASSWORD` | пароль приложения |

## Этап 4 — Деплой на Render (вы)

### 4.1. Создание сервиса

1. [dashboard.render.com](https://dashboard.render.com) → аккаунт знакомого
2. **New +** → **Blueprint** (если есть `render.yaml`)  
   **или** **Background Worker** вручную:
   - Runtime: **Python 3**
   - Build Command: `pip install -r requirements.txt`
   - Start Command: `python main.py`
   - Plan: **Starter** ($7/мес) — нужен для работы 24/7

> **Важно:** бесплатный тариф Render **не подходит** для постоянной работы воркера. Для 24/7 нужен платный Starter.

### 4.2. Переменные окружения (Environment)

В настройках сервиса → **Environment** → добавить:

```
TELEGRAM_BOT_TOKEN  = <токен от знакомого>
TELEGRAM_CHAT_ID    = <chat_id из get_chat_id.py>
EMAIL_HOST          = imap.gmail.com
EMAIL_PORT          = 993
EMAIL_USER          = <почта>
EMAIL_PASSWORD      = <пароль приложения>
```

### 4.3. Подключение GitHub

- **Settings** → Connect репозиторий
- При каждом `git push` в `main` Render автоматически пересобирает и перезапускает бота

### 4.4. Проверка

1. **Logs** в Render — должно быть: `Бот запущен. Активных модулей: 1`
2. Отправить тестовое письмо OLX на ящик (или дождаться реального)
3. В Telegram-чате должно появиться уведомление с кнопкой

## Этап 5 — Передача знакомому

Когда всё работает, передайте знакомому:

### Документы
- Ссылку на GitHub-репозиторий
- Ссылку на Render Dashboard (сервис `kirs-bot`)
- Этот файл `HANDOFF.md` и `README.md`

### Доступы (по желанию знакомого)
- GitHub: он владелец репозитория
- Render: он владелец аккаунта, вы можете отозвать свой доступ
- Telegram: бот у него в BotFather

### Что умеет бот
- Каждые 60 сек проверяет почту
- При новом письме OLX шлёт в чат: аккаунт, ссылку, кнопку «Перейти в чат»
- Работает 24/7 на сервере Render, ПК не нужен

### Как вносить изменения в будущем
1. Вы (или знакомый через вас) меняете код локально
2. `git push` → Render сам обновит бота
3. Настройки модулей — в `config.yaml` (фильтры, шаблон сообщения)

### Стоимость
- Render Starter Worker: ~$7/месяц
- Telegram и GitHub: бесплатно

## Частые проблемы

| Проблема | Решение |
|----------|---------|
| `TELEGRAM_BOT_TOKEN не задан` | Проверить Environment на Render, перезапустить сервис |
| Ошибка IMAP login | Использовать пароль приложения, не обычный пароль |
| Бот не пишет в группу | Проверить `TELEGRAM_CHAT_ID`, бот добавлен в группу |
| Письма не обрабатываются | Проверить `subject_filter` в `config.yaml`, посмотреть логи |
| Неверный email аккаунта в уведомлении | Прислать пример письма — подстроить парсер в `parser.py` |
